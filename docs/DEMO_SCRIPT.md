# 시연 대본 (5분)

준비: 배포 URL 접속(접속 코드 입력), 아래 파일 3개를 바탕화면에 준비.
- `data/samples/clean/clean_sticker.pdf` — 정상 스티커 (90x90, 칼선 포함)
- `data/samples/corpus/single_03_sticker_bleed.pdf` — 재단여백 없는 스티커
- `data/samples/corpus/single_11_sticker_trim_safety.pdf` — 재단선에 텍스트가 붙은 스티커

핸드폰 스톱워치를 하나 띄워두면 효과가 크다.

---

**0:00–0:40 | 문제 제기.** 화면 상단 "비교 데모" 토글을 켠다. 왼쪽에 지금
게시판에서 실제로 일어나는 왕복이 보인다 — 09:12 문의, 11:40 "파일 확인해보겠습니다",
16:55 "여백이 부족합니다, 수정 후 재업로드 부탁드립니다", 다음날 오전 확정.
**"주문 하나에 하루가 갑니다. 같은 일을 오른쪽에서 해보겠습니다."**

**0:40–2:00 | 정상 주문 30초 완주.** 스톱워치 시작. 오른쪽 채팅에
"스티커 500장 주문하려고요" 입력 → `clean_sticker.pdf` 드래그.
- 검판 카드: 12개 항목 전부 통과, "인쇄 가능한 파일이에요" 배너
- 사이즈 90x90 자동 인식(파일에서 읽음), 칼선 감지 → 도무송 자동 판단
- 견적 카드 표시 → "이대로 확정하기" 클릭 → 주문번호.
스톱워치 정지. **"질문 몇 번 받으셨나요? 이 고객은 한 번도 안 받았습니다.
파일이 답을 알고 있으면 AI는 묻지 않습니다."**

**2:00–3:30 | 문제 파일 — 자동 수리.** 새 상담 시작.
"스티커 500장이요" + `single_03_sticker_bleed.pdf` 업로드.
- 검판 카드에 빨간 항목: "재단 여백이 사방 3mm 필요한데 지금 파일은 0mm예요"
- **"기존엔 이 시점에 '수정 후 재업로드' 댓글이 달리고 하루가 갔습니다."**
- "자동 보정 적용" 클릭 → 전/후 비교 이미지 → 재검판 전부 통과 → 견적 → 확정.
**"고객은 포토샵을 다시 열 필요가 없습니다."**

**3:30–4:20 | 회색지대 — 사람에게 넘긴다.** 새 상담,
`single_11_sticker_trim_safety.pdf` 업로드.
- "재단선 바로 옆에 글자가 있어요. 의도하신 디자인일 수도 있어서 담당자 검토로 넘겼어요" +
  에스컬레이션 카드.
**"AI가 다 아는 척을 하지 않습니다. 판단이 안 서는 파일은 반드시 사람 검판 큐로 갑니다 —
잘못 재단된 실물이 나가는 것보다 백 배 쌉니다."**

**4:20–5:00 | 라이브 슬롯.** **"지금 갖고 계신 실제 주문 파일 하나 넣어보시죠."**
(합성 데모의 방어막 — 어떤 결과가 나와도 좋다. 통과하면 견적까지, 걸리면 그 이유를
한국어로 설명하는 것 자체가 셀링 포인트.)
마무리: "왼쪽 게시판의 하루와 오른쪽의 30초. 저희가 만든 건 챗봇이 아니라
**검판 엔진에 말을 붙인 것**입니다. 측정과 가격은 전부 결정론이고, 대화만 AI입니다."

---

예상 질문 대비:
- "우리 가격표랑 다른데?" → 공개가 근사치를 넣은 데모용 매트릭스. 실가격표 yaml 하나 교체로 끝.
- "우리 파일은 더 지저분한데?" → 그래서 라이브 슬롯이 있다. 반려 파일 몇 개를 받으면
  eval 파이프라인에 그대로 넣어 재현율을 측정해서 보여줄 수 있다 (manifest 구조 동일).
- "카카오톡으로 되나?" → 오케스트레이터가 채널 무관 설계라 카카오 비즈니스 채널 어댑터만
  붙이면 된다 (본개발 항목).
